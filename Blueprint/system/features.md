## 1. Dashboard (README.md)

See the current state at a glance.

> **What you can see on Dashboard:**
> - List of ongoing Voyages (Tasks, Projects, Areas)
> - GitHub Contribution Graph style activity visualization
>   - `■□□■■□□■□□□□■■□□□□■□□□■□□□□□` ← Monthly activity pattern
>   - See at a glance which days worked (■) vs not (□)
> - Monthly/total time investment (e.g., "Total: 114.8h since Sep 01")
> - Recent activity per Voyage

> **Auto-generated:**
> - README.md auto-generated by script (based on Time Management App data)
> - User doesn't manually edit

> **Implemented (v1.2.0):**
> - Sailing Orders integration
>   - Parses `| Deadline | Created | Order | Purpose |` from each voyage-plan.md
>   - `## Underway` section shows orders currently in progress
>   - `## Urgent Sailing Orders` section shows deadlines within 7 days
>   - Per-voyage Orders table with Status column (Underway/Plotted)

> **Implemented (v1.4.0):**
> - Voyage Map output (`_system/data/voyage-map.json`) — navigation data for other scripts
> - Script: `py _system/scripts/dashboard/generate.py`

> **Not implemented:**
> - Missing record alerts
> - Long-abandoned Voyage warnings

## 2. Passage Builder (Implemented v1.4.0)

AI-orchestrated passage creation with Calendar integration and metacognition awareness.

> **What it does:**
> - Resolves fuzzy user input to exact voyage info
> - Queries Google Calendar for matching events
> - Applies decision matrix based on event state and timing
> - Creates passage files with pre-filled plan (if available from Calendar)
> - Delivers metacognition awareness messages (e.g., "Would you like to write the plan first?")

> **Decision Matrix:**
>
> | Calendar Event | Plan | Timing | Action |
> |---|---|---|---|
> | Exists | Filled | Any | Plan → passage-plan.md |
> | Exists | Empty | Before | Suggest writing plan first |
> | Exists | Empty | During | Create empty passage |
> | Exists | Empty | After | Create empty passage |
> | Not exists | - | - | Create empty passage + offer event creation |

> **CONTRACT:** `_system/scripts/passage-builder/CONTRACT.md`

## 3. Passage Plan Observer (Implemented v1.4.0)

Automated skeleton injection into empty Calendar event descriptions.

> **What it does:**
> - Scans upcoming Calendar events (today ~ +7 days)
> - Filters `[Project]`/`[Area]` events only
> - Injects planning skeleton template into empty event descriptions
> - Reduces friction for the metacognition Planning stage

> **Skeleton Template:**
> ```
> ### Task
> - What will I do? (including approach)
> ### Time Estimate
> - How long will it take?
> - Why do I think so?
> ### Difficulty Estimate
> - How well can I do this?
> - Why do I think so? (including expected obstacles)
> ```

> **Script:** `py _system/scripts/passage-plan-observer/observer.py`
> **CONTRACT:** `_system/scripts/passage-plan-observer/CONTRACT.md`

## 4. Calendar Integration (Implemented v1.4.0)

Micro scripts for Google Calendar read/write operations.

> **Scripts:**
>
> | Script | Purpose | Credential Scope |
> |--------|---------|-----------------|
> | `calendar/query.py` | Check if event exists, timing, has plan | `calendar.readonly` |
> | `calendar/read-description.py` | Extract plan content from event description | `calendar.readonly` |
> | `calendar/create-event.py` | Create new Calendar event with skeleton | `calendar.events` (read+write) |

> **Minimum privilege:** Read scripts use dashboard's readonly credential. Write script uses observer's read+write credential.

## 5. Voyage Resolver (Implemented v1.4.0)

Fuzzy input resolution to exact voyage info.

> **What it does:**
> - Takes user input (e.g., "riipen") → resolves to exact voyage info
> - Returns: title, category, tier, path, folder, match type
> - Data source: `_system/data/voyage-map.json` (generated by Dashboard), filesystem scan as fallback
> - Match priority: exact folder → folder contains → title contains

> **Script:** `py _system/scripts/voyage/resolve.py "{input}"`

## 6. Raw Processing (Implemented v1.3.0)

Quick capture without categorization overhead.

> **What it does:**
> - `### Raw` section in log.md, passage-plan.md, passage-forecast-review.md
> - Write rough notes without thinking about structure
> - AI organizes into appropriate sections on request

> **Cross-file organization (passage-forecast-review.md):**
> - Time/Difficulty actuals → stay in current file
> - Reflection/Blockers/Next Steps/Notes → move to ../log.md

> **How to use:**
> - Write anything in Raw section
> - Say "organize raw" to AI
> - AI categorizes and clears Raw section

## 8. AI Scan (Not implemented)

Automatically detect principle violations and structure deviations.

> ### Security Scan (Sensitive Information Detection)
>
> **Problem:**
> - Accidentally write sensitive info in Logbook or Voyage Plan
> - Risk when making public or sharing later
>
> **Detection targets:**
> - API keys / tokens
> - Passwords
> - Email addresses
> - Phone numbers
> - Credit card numbers
> - Private URLs (internal systems)
> - Other personally identifiable information (PII)

> ### Scope Scan (Scope Deviation Detection)
>
> **Problem:**
> - Content outside this system's scope gets mixed in
> - Untrackable categories emerge
>
> **Detection targets:**
> - User-defined principle violations
> - Content outside system scope (finance management, relationship management, etc.)

> ### Structure Scan (Structure Deviation Detection)
>
> **Problem:**
> - Structure inconsistency due to human error
>
> **Detection targets:**
> - Voyage misclassified as Task
> - Missing required files (no voyage-plan.md, etc.)
> - Folder structure inconsistency
> - Missing required template sections
